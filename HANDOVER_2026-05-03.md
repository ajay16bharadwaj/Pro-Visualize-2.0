## What Was Done

### Bug: GSEApy enrichment returning no results (Activity Scoring tab → "From DE Results")

**Root cause (two-layer problem):**

1. **Gene names were never stored.**  
   `_build_anndata()` called `pg_df.set_index(protein_col)` and then immediately subsetted to sample columns, silently discarding all non-sample metadata columns (e.g. `Genes`, `Gene.Names`) from the PG matrix. So `adata.var` was always an empty DataFrame indexed by the protein ID.

2. **Wrong IDs sent to Enrichr.**  
   `run_gsea_enrichment()` built the gene list directly from the `protein` column of DE results, which are `adata.var_names` — i.e. whatever column was set as `Protein column` at upload (default: `Protein.Group`, which contains UniProt accessions like `Q9Y490`). Enrichr only understands gene symbols, so it returned an empty DataFrame.  
   Confirmed experimentally: `gp.enrichr()` returns `results.empty = True` when it receives unrecognised identifiers.

## Files Changed

### `visualizations/scp_visualizer.py`

**1. `_build_anndata()` — preserve metadata columns**
```
Before: var_df = pd.DataFrame(index=pg_df.index)
After:  meta_cols = [c for c in pg_df.columns if c not in set(used_samples)]
        var_meta = pg_df[meta_cols].copy() if meta_cols else pd.DataFrame(...)
        var_df = var_meta.reindex(pg_df.index)
```
Non-sample columns (`Genes`, `Gene.Names`, etc.) are now kept in `adata.var` so downstream steps can use them.

**2. `run_gsea_enrichment()` — 3-priority gene symbol resolution**
```
Priority 1: gene-name column already in adata.var  (e.g. "Genes" from PG matrix)
Priority 2: on-demand UniProt lookup for the significant DEPs only
Priority 3: split protein ID on ";" as last resort
```
Resolved symbols are stored back into `adata.var["Genes"]` so subsequent runs don't re-query.

**3. New methods for UniProt annotation**
- `_parse_uniprot_accession(protein_id)` — strips semicolons (`P12345;Q67890` → `P12345`) and isoform suffixes (`P12345-2` → `P12345`)
- `_fetch_uniprot_gene_names(accessions, batch_size=200)` — batches requests to `rest.uniprot.org/uniprotkb/search`, returns `{accession: gene_symbol}`
- `annotate_gene_names_from_uniprot()` — runs the full lookup for all proteins in `adata.var`, stores in `adata.var["Genes"]`, returns count mapped

### `modules/scp_module.py`

**1. Upload & QC tab — gene name status block**  
After data loads, the tab now checks for a gene name column:
- If found → caption showing which column is used
- If missing → warning + **"Fetch Gene Names from UniProt"** button (calls `annotate_gene_names_from_uniprot()` with a spinner; ~10–30 s for a typical dataset)

**2. Activity Scoring → From DE Results — live filter preview**  
A caption below the filter controls now shows in real time:
- How many protein groups pass the current adj-p / FC / direction filters
- A sample of the first 5 protein names
- Whether a gene-name column was detected (or a warning if not)

**3. "No results" warning — more actionable message**  
The generic "try relaxing filters" message now also explains the gene-name mismatch and points to the UniProt button.

## What Still Needs Attention / Known Gaps

| Item | Notes |
|---|---|
| **UniProt rate limits** | The REST API has no strict published rate limit for small batches, but very large proteomes (>5000 proteins) may be slow. `batch_size=200` is conservative; 500 is also fine. |
| **Protein ID format assumptions** | Code assumes UniProt accession format. Non-UniProt IDs (e.g. RefSeq, Ensembl) will silently return no gene names from UniProt. |
| **Gene names for activity scoring** | `compute_activity_scores()` matches against `adata.var_names` directly. If `var_names` are UniProt accessions, manually entered gene sets in the "Define Gene/Protein Sets" expander also won't match unless the user types accessions. Consider renaming `var_names` to gene symbols after the UniProt fetch, or adding a separate lookup. |
| **`de_all` unused variable** | `de_all = viz.get_de_results()` at line ~641 in `scp_module.py` is never used. Pre-existing, not introduced this session. |
| **Session persistence of gene names** | Gene names fetched via UniProt are stored in `adata.var` in memory. If the page is refreshed or the session restarts, the user must re-fetch. No persistent cache across sessions. |
