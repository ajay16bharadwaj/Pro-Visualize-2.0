## Bug 1: GSEApy Enrichment Returning No Results

**Root cause (two-layer problem):**

1. Non-sample metadata columns (`Genes`, `Gene.Names`, etc.) were silently dropped from `adata.var` during AnnData construction in `_build_anndata()`. After `set_index(protein_col)`, the code subsetted to sample columns and discarded everything else, so `adata.var` was always an empty DataFrame.

2. `run_gsea_enrichment()` sent protein group IDs directly to Enrichr (UniProt accessions like `Q9Y490`, or contaminant entries like `contam_sp|O43790|KRT86_HUMAN`). Enrichr only accepts gene symbols — confirmed experimentally: `gp.enrichr()` returns `results.empty = True` for unrecognised identifiers.

**Fixes:**

- **`_build_anndata()`**: Save all non-sample columns into `var_meta` before subsetting to sample columns. Set `var_df = var_meta.reindex(pg_df.index)` instead of an empty DataFrame. Metadata columns like `Genes` are now preserved in `adata.var`.

- **`run_gsea_enrichment()` — 3-priority gene symbol resolution:**
  1. Gene-name column already in `adata.var` (e.g. `Genes` preserved from PG matrix)
  2. On-demand UniProt REST API lookup for the significant DEPs only (lazy/on demand)
  3. Split protein ID on `;` as last resort

- **`_parse_uniprot_accession()`**: Extended to handle FASTA-style IDs with pipes:
  - `contam_sp|O43790|KRT86_HUMAN` → `O43790`
  - `sp|P12345|GENE_HUMAN` → `P12345`
  - `P12345;Q67890` → `P12345` (semicolon-separated groups)
  - `P12345-2` → `P12345` (isoform suffix)

- **UniProt batch size**: Reduced from 200 → 50 per request. UniProt GET API limit is ~75 accessions; 200 caused 400 errors. Batch size of 50 confirmed working.

## Bug 2: Gene Name Column Present but All Null

**Root cause:** The PG matrix had a `Genes` column header but all values were `None`. Two downstream bugs followed:

1. **UniProt button hidden**: The detection check only tested column *presence* (`c in viz.adata.var.columns`), not whether it had any real values. An empty `Genes` column suppressed the button.

2. **False "1331/1331 have a gene symbol" count**: `pandas.Series.astype(str)` converts `None`/`NaN` to the literal strings `"None"`/`"nan"`, which pass a `!= ""` check.

**Fix:** Added `_n_valid_genes(series)` helper that correctly excludes `None`, `nan`, `na`, and empty strings. Detection now requires both column presence AND `_n_valid_genes > 0`.

## Bug 3: UniProt Contaminant Entry Format (400 Error)

**Root cause:** The last batches of proteins contained contaminant entries formatted as `contam_sp|O43790|KRT86_HUMAN`. The `|` characters got URL-encoded as `%7C`, breaking the UniProt query.

**Fix:** `_parse_uniprot_accession()` now detects the pipe-delimited FASTA format and extracts the middle field (the actual accession).

## Bug 4: Activity Scores Always 0 Matched (Gene Symbol Mismatch)

**Root cause:** `compute_activity_scores()` matched genes against `adata.var_names` (UniProt accessions). Enrichment results contain gene symbols (`POLR2A`, `EIF4G1`, etc.). These never match. Additionally, the on-demand UniProt fetch in `run_gsea_enrichment()` only populated `adata.var["Genes"]` for the significant DEPs — not all 1331 proteins. So `sym_to_var` (the gene symbol → var_name map) was sparse, and pathway genes from non-DEP proteins (like `POLR2B`) couldn't be resolved.

**Fixes:**

- **`compute_activity_scores()`**: Added `_build_gene_symbol_map()` helper that builds `{gene_symbol: var_name}` from `adata.var`. Resolution now tries: (1) direct var_name match, (2) gene symbol lookup, with deduplication.

- **"Compute Activity Scores from Selected Pathways" button**: Automatically triggers full UniProt annotation for all proteins if the gene map covers <50% of `adata.n_vars`. Shows a spinner: `"Fetching gene names from UniProt for all N proteins first…"`. This ensures all pathway genes can be resolved, not just DEPs.

- **Live preview captions**: Show `"Gene names only resolved for X/N proteins so far. Full UniProt annotation will run automatically…"` when gene map is sparse, so the user understands the 0-match counts are expected pre-annotation.
